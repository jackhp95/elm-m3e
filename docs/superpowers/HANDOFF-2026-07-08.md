  (`[&_m3e-card[orientation=horizontal]>img[slot=header]]:max-w-[clamp(...)]`).
  Likely shared by other wide-media examples (list rows, banners).

## 7. Conventions & the verification bar (non-negotiables)

- **Regen (deterministic no-op when nothing changed):**
  `node elm-cem/bin/elm-cem.js --flags-from=docs/node_modules/@m3e/web/dist/custom-elements.json --config-from=config/slots.json --config-from=config/native-mdn.json --config-from=config/examples.generated.json --output=packages/m3e/src`
- **2-pass reproducibility loop:** a `config/slots.json` change → regen → `cd docs &&
  npm run build:assets` (converts examples, writes back `config/examples.generated.json`
  with newly-working examples) → **regen again** (embeds the new `@docs` blocks).
  Converges — a 2nd `build:assets` reproduces identical output. Then `build:reference`
  to sync `data/reference.json` (gitignored).
- **`build` vs `build:ci`:** `build` runs the full `build:assets` (examples-gen —
  needs the `elm-review-cem` sibling + the compiled library); `build:ci` **skips** it
  and uses the committed `docs/data/examples.json`. **Netlify uses `build:ci`**; CI's
  Docs job uses `build`. So after a library change, **regenerate + commit the examples
  data** or Netlify shows stale examples.
- **Verification bar (before merge):** `packages/m3e` compiles (`cd packages/m3e &&
  ../../docs/node_modules/.bin/elm make src/M3e.elm --output=/dev/null`);
  `cd docs && npm run build:ci` exit 0; local `elm-review` (needs the sibling
  `elm-review-cem` rules — clone it beside the repo if missing); the Playwright
  harness (`cd docs && npm run build`, `node scripts/serve-dist.mjs dist 1239 &`,
  then `BASE_URL=... npx playwright test`; CommonJS chromium import from
  `docs/node_modules/.pnpm/@playwright+test@1.61.1/...`, wait ~1600ms). `elm-format`
  only the hand-written IR core (`M3e/Node.elm`, `M3e/Element.elm`) — generated
  modules are intentionally not formatted. Push directly to `main` requires the
  user's session authorization (the guardrail blocks it otherwise).
- `config/` is at the **repo root**, not under `docs/`.
- `elm-review-cem` tests use a pinned runner:
  `npx -y elm-test@0.19.1-revision12 --compiler docs/node_modules/.bin/elm`.
- Run `elm-format` after any `.elm` edit (esp. scripted).
