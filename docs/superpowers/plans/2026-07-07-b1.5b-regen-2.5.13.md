## Background the implementer must hold

The elm-cem CLI (`elm-cem/bin/elm-cem.js`) preprocesses the CEM through a chain of arg
transforms before shelling to `elm-codegen run codegen/Generate.elm`. Each transform reads the
CEM via `--flags-from`, mutates the parsed JSON, writes a temp file (`writeTemp`), and rewrites
the `--flags-from` arg to point at the temp. Composition today (line 35):

```js
const args = injectRuntime(injectConfig(inlineTypeAliases(process.argv.slice(2))));
```

`inlineTypeAliases` is innermost (runs first). We add `reconcileTagNames` as a NEW innermost pass
so corrected tags are visible to every later pass and to the generator.

`Generate.elm` reads `.tagName` off class declarations and MERGES components with the same tagName
(`mergeSameTagName`, ~line 3044). The whole bug is two declarations with wrong colliding tagNames;
fixing the tagName makes the merge stop swallowing the two modules.

## Steps

### Step 1 — Add `reconcileTagNames` to `elm-cem/bin/elm-cem.js`

Follow the EXACT structure of `injectRuntime` (lines 112–152): locate the `--flags-from` arg
(supporting both `--flags-from=X` and `--flags-from X` forms), parse the CEM, mutate, `writeTemp`,
rewrite the arg, return the new argv. On any failure (no flag / parse error) return argv unchanged
so non-m3e / config-free invocations are untouched.

Mutation logic:
```
for each module m in cem.modules:
  for each export e in (m.exports || []) where e.kind === "custom-element-definition":
    const tag = e.name
    const declName = e.declaration && e.declaration.name
    if (!tag || !declName) continue
    // resolve the declaration the export registers
    const declModule = e.declaration.module  // may be undefined (same module)
    const target = findDeclaration(cem, declName, declModule)  // prefer declModule, else search all modules by name
    if (target && target.customElement && target.tagName !== tag) {
      console.log(`elm-cem: reconciled tagName ${JSON.stringify(target.tagName)} -> ${JSON.stringify(tag)} for ${declName}`)
      target.tagName = tag
    }
```

- `findDeclaration`: if `declModule` given, find `cem.modules.find(mm => mm.path === declModule)`
  then its declaration by name; else scan all modules' declarations for `name === declName`. Guard
  the same-name-in-two-modules case by preferring the declModule match.
- Only rewrite when `target.customElement` is truthy AND the value actually differs (keeps the pass
  a no-op for correct analyzers → clean logs, minimal churn).
- Add a concise comment block above the function explaining WHY (analyzer emits wrong `tagName` on
  class declarations for `m3e-stepper-next`/`m3e-fab-menu-item`; the `custom-element-definition`
  export is the registration truth; without this the generator merges them into their siblings).

Wire it: change line 35 to
```js
const args = injectRuntime(injectConfig(inlineTypeAliases(reconcileTagNames(process.argv.slice(2)))));
```

Match the surrounding code style (CommonJS, the existing helper idioms, comment density).
